# wind-turbine-scada-sim

A simulated wind farm SCADA system demonstrating real-time telemetry collection, time-series storage, operational dashboards, and basic anomaly detection — built to learn industrial monitoring concepts using modern open-source tools.

## DISCLAIMER

The project's scaffold is generated by Claude. As I am new to SCADA, let alone wind engineering, I don't know what I don't know. So I discussed with Claude about this project, and made the scaffold for me to finish the logic myself.

## Quick Start

```bash
git clone https://github.com/jotafauzanh/wind-turbine-scada-sim.git
cd wind-turbine-scada-sim
docker compose up --build
```

Once running:
- **Grafana**: http://localhost:3000 (admin / admin)
- **OPC-UA Server**: opc.tcp://localhost:4840
- **InfluxDB UI**: http://localhost:8086 (admin / adminpassword)

## Tech Stack

- **Python 3.12** — simulator, collector, anomaly detector
- **opcua-asyncio** — OPC-UA server and client
- **InfluxDB 2.x** — time-series storage
- **Grafana** — operational dashboards
- **Docker Compose** — single-command deployment

## What I'd Add in Production

Things deliberately left out to keep scope manageable, but that a real SCADA system would need:

- **Modbus gateway** — many field devices still speak Modbus RTU/TCP alongside OPC-UA
- **IEC 62443 network segmentation** — OT network isolated from IT via DMZ
- **Redundant SCADA servers** — hot standby failover for the collector and historian
- **Ignition or WinCC as HMI** — production-grade operator interface replacing Grafana
- **Role-based access control** — operator vs engineer vs admin permissions
- **Alarm management system** — ISA 18.2 compliant alarm prioritization and shelving
- **IEC 61850 / DNP3** — substation and grid operator communication protocols
- **Edge computing** — local processing at the turbine before sending to central SCADA
- **Secure remote access** — VPN / jump server for remote troubleshooting
