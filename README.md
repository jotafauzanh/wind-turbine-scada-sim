# wind-turbine-scada-sim

A simulated wind farm SCADA system demonstrating real-time telemetry collection, time-series storage, operational dashboards, and basic anomaly detection вҖ” built to learn industrial monitoring concepts using modern open-source tools.

## DISCLAIMER

The project's scaffold is generated by Claude. As I am new to SCADA, let alone wind engineering, I don't know what I don't know. So I discussed with Claude about this project, and made the scaffold for me to finish the logic myself.

## Development Progress

### Phase 1

Phase 1 focuses on developing an MVP by creating the monitoring part with a funcitoning dashboard.

1. [x] OPC-UA Server (`simulator/opcua_server.py`)
2. [x] Turbine Physics (`simulator/turbine.py`)
3. [x] OPC-UA Client + Subscription (`collector/opcua_client.py`)
4. [x] InfluxDB Writer (`collector/influx_writer.py`)

## Quick Start

```bash
git clone https://github.com/jotafauzanh/wind-turbine-scada-sim.git
cd wind-turbine-scada-sim
docker compose up --build
```

Once running:
- **Grafana**: http://localhost:3000 (admin / admin)
- **OPC-UA Server**: opc.tcp://localhost:4840
- **InfluxDB UI**: http://localhost:8086 (admin / adminpassword)

## Tech Stack

- **Python 3.12** вҖ” simulator, collector, anomaly detector
- **opcua-asyncio** вҖ” OPC-UA server and client (https://opcua-asyncio.readthedocs.io/en/latest/index.html)
- **InfluxDB 2.x** вҖ” time-series storage
- **Grafana** вҖ” operational dashboards
- **Docker Compose** вҖ” single-command deployment

## What I'd Add in Production

Things deliberately left out to keep scope manageable, but that a real SCADA system would need:

- **Modbus gateway** вҖ” many field devices still speak Modbus RTU/TCP alongside OPC-UA
- **IEC 62443 network segmentation** вҖ” OT network isolated from IT via DMZ
- **Redundant SCADA servers** вҖ” hot standby failover for the collector and historian
- **Ignition or WinCC as HMI** вҖ” production-grade operator interface replacing Grafana
- **Role-based access control** вҖ” operator vs engineer vs admin permissions
- **Alarm management system** вҖ” ISA 18.2 compliant alarm prioritization and shelving
- **IEC 61850 / DNP3** вҖ” substation and grid operator communication protocols
- **Edge computing** вҖ” local processing at the turbine before sending to central SCADA
- **Secure remote access** вҖ” VPN / jump server for remote troubleshooting
